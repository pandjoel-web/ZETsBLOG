Apa Itu SQL?

SQL (Structured Query Language) adalah bahasa standar untuk mengelola dan memanipulasi basis data relasional. SQL memungkinkan Anda melakukan operasi seperti membuat, membaca, memperbarui, dan menghapus data (CRUD) pada sistem manajemen basis data relasional (RDBMS) seperti MySQL, PostgreSQL, atau SQLite.

Struktur Basis Data Relasional

Basis data relasional terdiri dari tabel yang menyimpan data dalam format baris dan kolom. Setiap tabel memiliki:

  • Kolom: Atribut atau field (misalnya, nama, umur).
  • Baris: Catatan atau entri data individu.
  • Kunci Primer (Primary Key): Kolom unik untuk mengidentifikasi setiap baris (misalnya, id).

Contoh tabel pelanggan:

id nama umur kota
1 Pandu 18 Solo
2 Gesya 18 heaven

Perintah Dasar SQL

1. Membuat Tabel (CREATE TABLE)

Untuk membuat tabel baru:


CREATE TABLE pelanggan (
    id INT PRIMARY KEY,
    nama VARCHAR(100),
    umur INT,
    kota VARCHAR(50)
);

2. Menambahkan Data (INSERT INTO)

Untuk menambahkan data ke tabel:


INSERT INTO pelanggan (id, nama, umur, kota)
VALUES (1, 'Pandu', 18, 'Solo');

3. Mengambil Data (SELECT)

Untuk mengambil data dari tabel:


-- Mengambil semua kolom
SELECT * FROM pelanggan;

-- Mengambil kolom tertentu
SELECT nama, kota FROM pelanggan;

4. Memfilter Data (WHERE)

Untuk memfilter data berdasarkan kondisi:


SELECT * FROM pelanggan WHERE umur > 17;

5. Memperbarui Data (UPDATE)

Untuk memperbarui data yang sudah ada:


UPDATE pelanggan
SET kota = 'heaven'
WHERE id = 1;

6. Menghapus Data (DELETE)

Untuk menghapus data:


DELETE FROM pelanggan WHERE id = 2;

Pengelompokan dan Pengurutan

1. Mengurutkan Data (ORDER BY)

Untuk mengurutkan hasil query:


SELECT * FROM pelanggan ORDER BY umur DESC;

2. Mengelompokkan Data (GROUP BY)

Untuk mengelompokkan data, sering digunakan dengan fungsi agregasi seperti COUNT, SUM, atau AVG:


SELECT kota, COUNT(*) as jumlah
FROM pelanggan
GROUP BY kota;

üí°Tips Belajar SQL

  • Eksplorasi: Coba query sederhana, lalu tambahkan kompleksitas seperti JOIN untuk menghubungkan tabel.
